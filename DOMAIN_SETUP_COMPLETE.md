## Understanding Vercel Domains

### Preview Deployments
- **Format**: `project-name-git-branch-username-projects.vercel.app`
- **Auto-generated**: Created automatically for each branch/deployment
- **Purpose**: Preview/test deployments before production
- **Current**: `cursor-gtm-training-git-main-andrewkosel93-1443s-projects.vercel.app`

### Production Domain
- **Format**: `project-name.vercel.app`
- **Default**: `cursor-gtm-training.vercel.app`
- **Purpose**: Main production URL
- **Status**: May need to be set as production deployment

### Custom Domain
- **Format**: Your own domain (e.g., `cursor-gtm-enablement-trainer.com`)
- **Purpose**: Professional branded URL
- **Status**: Not configured yet

## Setting Up Custom Domain

### Step 1: Add Domain in Vercel

1. Go to **Vercel Dashboard**
   - https://vercel.com/dashboard
   - Click your project: **cursor-gtm-training**

2. Navigate to **Settings ‚Üí Domains**

3. Click **Add Domain**

4. Enter: `cursor-gtm-enablement-trainer.com`

5. Click **Add**

6. Vercel will show DNS configuration instructions

### Step 2: Configure DNS

At your domain registrar, add:

**Option A: A Record**
- **Type**: `A`
- **Name**: `@` (or root)
- **Value**: `76.76.21.21`
- **TTL**: `3600` (or default)

**Option B: CNAME Record**
- **Type**: `CNAME`
- **Name**: `@` (or root)
- **Value**: `cname.vercel-dns.com`
- **TTL**: `3600` (or default)

### Step 3: Wait for DNS Propagation

- **Time**: 15 minutes to 48 hours
- **Usually**: 1-2 hours
- **Check**: Run `./check-domain-status.sh`

### Step 4: Verify

Once DNS propagates:
- Domain will show "Valid" in Vercel Dashboard
- SSL certificate will be auto-provisioned
- Site will be accessible at `https://cursor-gtm-enablement-trainer.com`

## Setting Production Domain

To make `cursor-gtm-training.vercel.app` work:

1. Go to **Deployments** tab
2. Find your latest successful deployment
3. Click **...** (three dots)
4. Click **Promote to Production**

This will make it the default production deployment.

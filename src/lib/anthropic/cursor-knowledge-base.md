# Browserbase Knowledge Base for AI Training

## Company Overview

**Browserbase** is a browser automation and web scraping platform designed to help engineering teams scale their browser workflows without managing infrastructure. It provides managed headless browser infrastructure, testing capabilities, and web scraping tools.

### Mission
To make browser automation accessible and scalable for engineering teams of all sizes through managed infrastructure.

### Target Audience
Engineering teams, developers, and technical teams looking to scale browser automation and web scraping workflows.

### Medium Importance
- **Pace** (140-180 WPM): Speaking speed

## How to Use This Context

When providing voice coaching feedback to Browserbase users:

1. **Relate to Developer Scenarios**: Reference code reviews, demos, standups
2. **Emphasize Professional Growth**: Connect improvements to career advancement
3. **Highlight Team Impact**: Show how better communication helps teams
4. **Reference Technical Context**: Mention code, architecture, and technical concepts
5. **Focus on Productivity**: Connect voice improvements to work efficiency
6. **Consider Remote Work**: Address distributed team communication needs

## Example Feedback Context

Instead of generic feedback like "Your pace is good", provide context-aware feedback like:

"Your speaking pace of 160 WPM is excellent for technical presentations. This speed allows you to clearly explain code architecture while keeping your audience engaged during code reviews and demos. Maintaining this pace will help you effectively communicate complex technical concepts to your team."
